================================================================================
ARCHITECTURE PATTERNS
================================================================================

1. Modular Commands
   - Each command is independent script
   - All follow same pattern: parse args → check env → execute → log
   - Easy to add new commands (3 steps)

2. Shared Libraries
   - colors.sh (37 lines) - ANSI colors and symbols
   - helpers.sh (227 lines) - Core utilities and logging
   - services.sh (97 lines) - Service definitions and mappings

3. Service Definition
   - Defined as bash arrays
   - Three categories: infrastructure, application, frontend
   - Single source of truth for service info

4. Logging System
   - Three-tier: console + file + environment variable
   - Automatic timestamped files: scripts/logs/dev-{command}-{timestamp}.log
   - Every command logs

5. UI/UX
   - Colors: GREEN (success), RED (error), YELLOW (warn), BLUE (info)
   - Symbols: ✓ (running), ✗ (failed), ⚠ (warning)
   - Formatted headers and progress indicators

================================================================================
TROUBLESHOOTING QUICK GUIDE
================================================================================

Problem: Docker not running
Solution: Start Docker desktop, then retry command

Problem: Service won't start
Solution: Check logs (dev logs [service]), verify health (dev health)

Problem: Port already in use
Solution: dev stop --hard, then dev start

Problem: Need to reset
Solution: dev reset (keeps data), or dev reset --rebuild (full wipe)

Problem: Want to debug
Solution: dev shell [service], then debug directly

Problem: Need test report
Solution: dev e2e-test --report

For more help: dev help
