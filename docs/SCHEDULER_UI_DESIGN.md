# Scheduler UI Design - Dashboard Integration

**Date:** 2025-11-22
**Session:** #89
**Status:** ğŸ¨ Design Complete

---

## Overview

Comprehensive UI design for managing and monitoring scheduled jobs in the Agentic SDLC dashboard. Provides intuitive interfaces for creating jobs, monitoring executions, analyzing performance, and managing event handlers.

---

## Design Principles

1. **Progressive Disclosure** - Show simple view by default, reveal complexity on demand
2. **Real-Time Updates** - Live status updates via WebSocket
3. **Visual Hierarchy** - Critical information prominent, details accessible
4. **Action-Oriented** - Quick actions always visible
5. **Responsive Design** - Mobile-first, works on all screen sizes

---

## Page Structure

### Navigation (Sidebar Addition)

```tsx
<NavItem icon={Clock} to="/scheduler">
  Scheduler
  <Badge variant="info">{activeJobsCount}</Badge>
</NavItem>
```

**Sub-navigation:**
- `/scheduler/jobs` - Job Management (default)
- `/scheduler/executions` - Execution History
- `/scheduler/events` - Event Handlers
- `/scheduler/analytics` - Performance Analytics

## API Integration

### API Client Methods

```typescript
// Jobs
export const schedulerApi = {
  // Jobs
  createCronJob: (data: CreateCronJobInput) =>
    POST('/api/v1/scheduler/jobs/cron', data),

  createOneTimeJob: (data: CreateOneTimeJobInput) =>
    POST('/api/v1/scheduler/jobs/once', data),

  getJob: (jobId: string) =>
    GET(`/api/v1/scheduler/jobs/${jobId}`),

  listJobs: (filters: JobFilters) =>
    GET('/api/v1/scheduler/jobs', { params: filters }),

  updateSchedule: (jobId: string, schedule: string) =>
    PUT(`/api/v1/scheduler/jobs/${jobId}/schedule`, { schedule }),

  pauseJob: (jobId: string) =>
    POST(`/api/v1/scheduler/jobs/${jobId}/pause`),

  resumeJob: (jobId: string) =>
    POST(`/api/v1/scheduler/jobs/${jobId}/resume`),

  deleteJob: (jobId: string) =>
    DELETE(`/api/v1/scheduler/jobs/${jobId}`),

  // Executions
  getJobHistory: (jobId: string, options: HistoryOptions) =>
    GET(`/api/v1/scheduler/jobs/${jobId}/executions`, { params: options }),

  getExecution: (executionId: string) =>
    GET(`/api/v1/scheduler/executions/${executionId}`),

  retryExecution: (executionId: string) =>
    POST(`/api/v1/scheduler/executions/${executionId}/retry`),

  // Events
  triggerEvent: (data: TriggerEventInput) =>
    POST('/api/v1/scheduler/events/trigger', data),
};
```
